Seção 1

Visão Geral

O QBank é uma aplicação web para estudo pessoal de concursos públicos, focada inicialmente em Contabilidade Geral. O sistema permite praticar questões de bancas reais organizadas por aula, fixar questões para revisão, realizar simulados cronometrados e consultar o histórico de desempenho.

Todo o conteúdo (questões e teoria) é extraído de PDFs de aulas e armazenado como arquivos JSON estáticos no próprio repositório. Não há backend de aplicação — a lógica roda inteiramente no navegador do usuário.

Princípio de design: zero dependências de build. Sem webpack, sem transpilador, sem npm no front-end. O código é HTML + CSS + JavaScript puro (ES6+), carregado diretamente pelo navegador.

Acesso

AmbienteURL
Produçãohttps://qbank-v01.web.app
Alternativohttps://qbank-v01.firebaseapp.com

Requisitos de uso

  • Navegador moderno (Chrome, Firefox, Safari, Edge)
  • Conta Google para autenticação
  • Conexão com internet (Firebase Auth + Firestore)

Por que sem framework? O app é essencialmente uma SPA simples de leitura de conteúdo. Vanilla JS mantém o código legível, o bundle zero e a manutenção direta — sem abstração desnecessária para a complexidade do problema.
Seção 3

Arquitetura de Navegação

A interface é organizada em quatro camadas de navegação em cascata, todas com posicionamento sticky — fixam no topo à medida que o usuário rola a página.

Início Fixadas Simulado Histórico ← topbar (abas globais) — z:100 ───────────────────────────────────────────────── ContG ContC ... ← barra de matérias — z:99 ───────────────────────────────────────────────── Aula 00 Aula 01A Aula 01B ... ← barra de aulas — z:98 ───────────────────────────────────────────────── [000] [000] [000] Expandir tudo ← barra de placar — z:10

3.1 Topbar — abas globais

Barra superior fixa em top: 0, altura de 52px (48px em mobile). Contém:

  • Nome do app (QBank) — oculto em telas < 600px
  • Abas globais: Início · Fixadas · Simulado · Histórico
  • Avatar e botão Sair (logout)
AbaComportamento ao clicar
Início Sempre reseta para a primeira matéria + primeira aula; recarrega a lista de questões
Fixadas Exibe questões fixadas; pré-seleciona a primeira aula como filtro ativo
Simulado Abre a tela de configuração do simulado; oculta barras de matéria e aula
Histórico Lista simulados passados ordenados por data; oculta barras de matéria e aula
Ao navegar para qualquer aba global durante um simulado ativo, o temporizador é limpo (clearInterval) e o estado do simulado é descartado.

3.2 Barra de matérias

Posicionada em top: 52px, altura de 42px, fundo cinza-claro #f0f0f0. Exibida em todas as abas. Botões com font-weight: 700; aba ativa indicada por underline preto na borda inferior (sem preenchimento).

Ao clicar em uma matéria: reseta para a primeira aula daquela matéria, limpa o filtro de fixadas e ativa a aba Início.

3.3 Barra de aulas

Posicionada em top: 94px, altura de 42px, fundo branco-acinzentado #fafafa. Visível apenas nas abas Início e Fixadas; oculta em Simulado e Histórico. Botões com font-weight: 600; mesma lógica de underline da barra de matérias.

Na aba Fixadas, clicar numa aula ativa ou desativa o filtro por aula (toggle). Na aba Início, carrega as questões daquela aula.

3.4 Barra de placar (sticky)

Dentro da área de conteúdo, posicionada em top: 136px (modo lista) ou top: 94px (modo simulado). Contém três chips coloridos e o botão Expandir tudo.

ChipCorConteúdo
000 Total Azul #dbeafe / #1d4ed8 Total de questões da aula
000 Acertos Verde #dcfce7 / #15803d Questões respondidas corretamente na sessão
000 Erros Vermelho #fee2e2 / #b91c1c Questões respondidas incorretamente na sessão

Os números são exibidos com 3 dígitos zero-padded (ex.: 007) em fonte monoespaçada. O progresso é apenas da sessão atual — nunca gravado.

Seção 4

Funcionalidades

4.1 Modo de questões (lista)

Ao selecionar uma aula no Início, o app carrega o JSON correspondente via fetch() e renderiza todas as questões em uma lista vertical de cards. O resultado do JSON é cacheado em memória (aulaCache) para evitar requisições repetidas na sessão.

Cada card exibe, em ordem:

  1. Meta: número da questão (Q1, Q2…), dificuldade em estrelas (★★☆☆☆), tipo e botão Fixar
  2. Banca (cinza, acima do enunciado) — ex.: (FGV/PC-AM/2022)
  3. Enunciado com suporte a diagramas em canvas
  4. Opções de resposta (botões interativos)
  5. Botão Ver gabarito → expande gabarito inline

4.2 Tipos de questão

Múltipla escolha

Até 5 alternativas (A–E). Ao clicar em uma opção: todas ficam desabilitadas; a correta fica verde verde, a escolhida errada fica vermelha vermelha. O placar é atualizado imediatamente.

Certo / Errado

Dois botões: A) Certo e B) Errado. Mesmo comportamento visual da múltipla escolha após resposta.

Expandir / Recolher tudo

Botão no canto direito da barra de placar. Alterna entre mostrar e ocultar o gabarito de todas as questões de uma vez.

4.3 Botão Fixar / Fixada

Cada questão tem um botão de marcação na linha de meta. Comportamento visual:

EstadoVisualAção ao clicar
Fixar Borda amarela #f59e0b, texto amarelo; hover fundo claro Adiciona a questão às Fixadas
Fixada Fundo amarelo sólido, texto branco; hover amarelo escuro #d97706 Remove a questão das Fixadas

Ao fixar: a questão entra em revisaoIds (Set em memória) e é gravada no Firestore (usuarios/{userId}/revisao/{questaoId}). A operação do Firestore é fire-and-forget — a UI responde imediatamente sem esperar a confirmação do banco.

São armazenados junto à questão os metadados de origem: _materia, _aula, _slug e _qNum (número original na aula).

4.4 Aba Fixadas

Exibe todas as questões marcadas usando o cache local (revisaoQuestoes[]), carregado do Firestore no login. A resposta é instantânea — não há nova requisição ao banco ao entrar na aba.

  • A barra de aulas funciona como filtro por aula de origem (toggle)
  • A meta de cada questão exibe a matéria e aula de origem
  • Ao desmarcar uma questão nesta aba, o card é removido do DOM imediatamente
  • O número original da questão (_qNum) é preservado no indicador Qn

4.5 Simulado

Configuração

O usuário escolhe dois parâmetros:

  • Fonte: matéria inteira ou uma aula específica
  • Quantidade: 10, 20 ou 30 questões

As questões são sorteadas aleatoriamente do pool disponível. Se o pool for menor que a quantidade solicitada, usa-se o total disponível.

Fluxo da questão

Uma questão por vez. Ao responder: o gabarito aparece imediatamente (✓ Correto / ✗ Incorreto) com o comentário, e o botão Próxima → avança para a seguinte. Na última questão, o botão vira Ver resultado.

A barra sticky do simulado exibe: 000 total · 000 acertos · 000 erros · cronômetro crescente à direita.

O cabeçalho da questão não exibe matéria/aula durante o simulado (para não dar pistas ao estudante).

Resultado

Ao finalizar: placar n / total, percentual de acerto, tempo total e gabarito completo com todas as questões e respostas. O resultado é salvo automaticamente no Firestore. O botão Novo Simulado retorna à tela de configuração.

4.6 Histórico

Lista todos os simulados realizados, ordenados do mais recente ao mais antigo. Cada item exibe: data, total de questões, tempo e placar (n/total + percentual).

Clicar em um simulado abre o gabarito completo daquele simulado, com todas as questões, respostas dadas e respostas corretas. O botão ← Histórico retorna à lista.

O botão Limpar histórico apaga todos os registros do Firestore (pede confirmação antes de executar).

4.7 Diagramas em canvas

Questões de Contabilidade frequentemente incluem tabelas/diagramas de Balanço Patrimonial desenhados com caracteres box-drawing Unicode (┌ ┐ └ ┘ │ ─ ┬ ┴ ┼ ├ ┤).

O app detecta automaticamente linhas com esses caracteres no enunciado e os converte para um elemento <canvas> com renderização precisa:

  • Linhas de borda desenhadas pixel a pixel com a Canvas API
  • Texto dos campos centralizado vertical e horizontalmente dentro de cada célula
  • Fundo cinza-claro #f9fafb, borda #e5e7eb
  • Responsivo: max-width: 100% com aspect-ratio preservado
  • Suporte a DPI alto (devicePixelRatio) para telas Retina
Seção 5

Estrutura de Dados

5.1 Arquivos JSON de conteúdo

Cada aula é um arquivo JSON em data/{materia}/{slug}.json. Atualmente: data/contabilidade/. Os arquivos são servidos como assets estáticos pelo Firebase Hosting.

Estrutura raiz do arquivo

{
  "slug":    "aula-01a",
  "titulo":  "Aula 01A — Balanço Patrimonial",
  "materia": "Contabilidade Geral",
  "teoria":  "## Título\n\nTexto em Markdown...",
  "questoes": [ ... ]
}

5.2 Schema de uma questão

Múltipla escolha

{
  "id":         "cg-01a-07",
  "banca":      "FGV/PC-AM/Investigador/2022",
  "tipo":       "multipla_escolha",
  "enunciado":  "Texto da questão...",
  "opcoes": [
    "A) Primeira alternativa",
    "B) Segunda alternativa",
    "C) Terceira alternativa",
    "D) Quarta alternativa",
    "E) Quinta alternativa"
  ],
  "resposta":    "D",
  "comentario":  "Explicação da resposta correta...",
  "dificuldade": 3
}

Certo / Errado

{
  "id":         "cg-01a-12",
  "banca":      "CESPE/TCU/2023",
  "tipo":       "certo_errado",
  "enunciado":  "Afirmação a ser julgada...",
  "resposta":   "certo",
  "comentario": "Explicação da resposta...",
  "dificuldade": 2
}

Nota: o campo opcoes está presente somente em multipla_escolha.

Tabela de campos

CampoTipoValoresDescrição

bancastringlivreBanca/concurso de origem; exibido em cinza
tipostring"multipla_escolha" | "certo_errado"Tipo da questão
enunciadostringtexto livreTexto da questão; suporta quebras de linha e diagramas Unicode

Convenção de IDs

Formato: {materia_abreviada}-{numero_aula}-{numero_questao}

  • cg = Contabilidade Geral
  • 00, 01a, 01b = número da aula
  • 01, 02… = número sequencial da questão

Exemplos: cg-00-01, cg-01a-07, cg-02b-03

5.3 Estrutura do Firestore

Todos os dados do usuário vivem sob o documento do próprio usuário, isolados por UID:

usuarios/
  {userId}/
    perfil/
      dados            → { nome, email, fotoUrl, criadoEm }

    historico/
      {simuladoId}     → {
                           data,           // Timestamp
                           fonte,          // ex: "aula:contabilidade:aula-01a"
                           placar,         // acertos
                           total,          // total de questões
                           tempoSegundos,
                           questoes[]      // snapshot completo das questões
                         }

    revisao/
      {questaoId}      → {
                           id, banca, tipo, enunciado,
                           opcoes,        // null para certo_errado
                           resposta, comentario, dificuldade,
                           _materia,      // nome da matéria (ex: "ContG")
                           _materiaId,    // slug (ex: "contabilidade")
                           _aula,         // título da aula (ex: "Aula 01A")
                           _slug,         // slug da aula (ex: "aula-01a")
                           _qNum,         // número original na aula
                           marcadoEm      // Timestamp
                         }
O campo questoes[] no histórico armazena um snapshot completo — as questões ficam registradas mesmo que o arquivo JSON fonte seja alterado posteriormente.
Seção 6

Regras de Segurança

O Firestore é protegido por regras que garantem isolamento total entre usuários: cada usuário só consegue ler e escrever nos próprios documentos. 

rules_version = '2';
service cloud.firestore {
  match /databases/{database}/documents {
    match /usuarios/{userId}/{document=**} {
      allow read, write: if request.auth != null
                         && request.auth.uid == userId;
    }
  }
}

A regra impõe duas condições simultâneas:

  1. O usuário deve estar autenticado (request.auth != null)
  2. O UID do usuário autenticado deve ser igual ao userId do documento acessado

Para aplicar alterações nas regras: 

firebase deploy --only firestore:rules
API Key pública: a apiKey do Firebase no app.js é uma chave pública — não é um segredo. A segurança real é garantida pelas Firestore Rules acima, não pela obscuridade da chave.
Seção 7

Autenticação

O app usa Firebase Authentication com provedor Google (OAuth 2.0 via popup). O estado de autenticação é monitorado por auth.onAuthStateChanged(), que é o ponto de entrada de toda a inicialização do app.

Fluxo de login

  1. App carrega → onAuthStateChanged dispara
  2. Se não autenticado: exibe tela de login (cartão centralizado com botão Google)
  3. Usuário clica em Entrar com Google → popup OAuth
  4. Após autenticação bem-sucedida: onAuthStateChanged dispara novamente
  5. Se autenticado: oculta tela de login, exibe o app
  6. Salva/verifica perfil no Firestore (perfil/dados — criado apenas se não existir)
  7. Carrega as questões fixadas (carregarRevisao())
  8. Renderiza o conteúdo inicial (primeira matéria, primeira aula)

Logout

Botão Sair no canto superior direito chama auth.signOut(), que dispara novamente o onAuthStateChanged e volta para a tela de login. Todo o estado em memória é descartado automaticamente.

Seção 9

Como Adicionar Novo Conteúdo

O fluxo padrão para adicionar uma nova aula ao banco de questões:

Passo 1 — Extrair o PDF

pdftotext -enc UTF-8 ContG_0200.pdf ContG_0200.txt

Passo 2 — Estruturar o JSON

Criar o arquivo em data/contabilidade/aula-XX.json seguindo o schema da Seção 5.

  • Questões extraídas da seção "Lista de Questões" do PDF
  • Comentários extraídos da seção "Questões Comentadas"
  • Campo banca separado do enunciado — nunca embutir no texto
  • Atribuir dificuldade de 1 a 5 para cada questão
  • Mínimo de 30 questões por aula
  • Teoria: síntese estruturada em Markdown (não cópia fiel; cobre definições, classificações, tabelas)

Passo 3 — Registrar no app.js

Adicionar a entrada na constante MATERIAS no início de js/app.js: 

const MATERIAS = [
  {
    id: 'contabilidade',
    nome: 'ContG',
    aulas: [
      { slug: 'aula-00',  titulo: 'Aula 00' },
      { slug: 'aula-01a', titulo: 'Aula 01A' },
      { slug: 'aula-02a', titulo: 'Aula 02A' },  // ← nova entrada
      // ...
    ]
  }
];
Importante: o slug deve corresponder exatamente ao nome do arquivo JSON sem a extensão. Ex.: "aula-02a" → data/contabilidade/aula-02a.json.

Passo 4 — Commitar e publicar

git add data/contabilidade/aula-02a.json js/app.js
git commit -m "content: adicionar Aula 02A — [Título]"
git push origin master
firebase deploy --only hosting

Adicionando nova matéria

Para adicionar uma matéria diferente de Contabilidade:

  1. Criar pasta data/{slug-da-materia}/
  2. Adicionar os arquivos JSON seguindo o mesmo schema
  3. Registrar novo objeto em MATERIAS com id único, nome (abreviação) e lista de aulas
Seção 10

Deploy e Configuração

Fluxo de publicação

Todo trabalho finalizado deve seguir este fluxo — Git e Firebase Hosting são independentes: 

git add [arquivos alterados]
git commit -m "tipo: descrição da mudança"
git push origin master          ← publica no GitHub
firebase deploy --only hosting  ← publica no site ao vivo

Configuração do Firebase Hosting (firebase.json)

  • Raiz pública: "." (diretório raiz do projeto)
  • Ignorados no deploy: firebase.json, arquivos ocultos (**/.*), node_modules/, PDFs/
  • Cache HTTP: no-cache, no-store, must-revalidate em .js, .css e index.html — garante que o navegador sempre baixa a versão mais recente
Cache busting manual: os ativos JS e CSS usam parâmetro ?v=2 na URL de carregamento (app.js?v=2, style.css?v=2). Incrementar este número força navegadores que ignoram os headers a baixar a versão nova.

Deploy apenas das regras do Firestore

firebase deploy --only firestore:rules

Deploy completo (hosting + rules)

firebase deploy
